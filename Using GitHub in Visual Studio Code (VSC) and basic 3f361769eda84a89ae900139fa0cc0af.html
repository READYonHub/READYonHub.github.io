Using GitHub in Visual Studio Code (VSC) and basic settings

  • Clone repository

    1. Open an integrated terminal from Terminal -> New Terminal (Ctrl + Shift + ö).

    1. Clone your repo with the following git command:
    git clone https://github.com/YOUR-NAME-OR-ORGANIZATION/YOUR-REPO-NAME
    1. Change your terminal into that new subdirectory:
    cd YOUR-REPO-NAME

    You can chech what files do you have with dir command:

    1. Then open in Visual Studio Code:
    code .
  • Github használata

    Rövid bevezető

    A Git egy elosztott verziókövető rendszer. Angolul Decentralized Version Control System, röviden DVCS, de Distributed Version Control System rövidítésének is tartják. Ez azt jelenti, hogy a felhasználó nem csak az aktuális verziót tartja a gépén, hanem az egész tároló (repository) tartalmát, ellentétben a központosított rendszerekkel. Ennek az az előnye, hogy hálózat nélkül is tudunk dolgozni (offline).

    Müködése: a gitben, amikor fájlokon és más erőforrásokon dolgozunk, minden helyben történik, nincs  szükség hálózati kapcsolatra . Így a munkánk gyors és offline lehet. A munkánkat bármikor szinkronizálhatjuk egy központi tárolóval.

    GitHub-Regisztráció

    Szükségünk lesz egy:

    Username: felhasználóneved

    Email: emailod@gmail.com

    Password: jelszavad123

    Ezután kapunk egy email-t is, abban egy kóddal, arra rá klikkelünk és onnantól van kész a teljesértékü regisztrációnk.

    Weboldalunk megosztása

    Ha megakarjuk osztani a világgal az oldalunkat, akkor szükségünk lesz egy helyre, ahol frltölthetjük az oldalunk forráskódját, ezek az úgy nevezett repository-k.

    Új repository-t, úgy tudunk létrehozni, ha bejelentkezünk a github fiókunkba, és jobb felül a plusz (+) jelre kattintunk és New repository.

    Ahhoz, hogy megtudjuk osztani a munkákat követnünk kell néhány szabályt, amit a GitHub Pages kapcsán ad nekünk a GitHub.

    Ez röviden annyi, hogy ha az oldalon leírt szabályokat (https://pages.github.com/) betartjuk, akkor a világ számara elérhetővé válik a tartalmunk.

    A lenti képen látható formában kell létrehoznunk a repositorynkat, ha weboldalt akarunk megosztani.

    Az Owner alatt található a fiók nevünk, a Repository name az a domain név lesz, amire ha rákeresünk a böngészőnkben visszakapjuk a weboldalunkat (RepositoryName.github.io). Fontos, hogy kisbetüvel írjuk be az Owner nevünket.

    Ne hadjuk ki, hogy bepipáljuk a Public opciót, hogy látható legyen mindenki számára.

    Ha ezek készen vannak click a Create repository gombra legalul.

    Weboldalunk feltöltése

    Nyissuk meg a Visual Studio Code-ot. Én személy szerint előnyben részesítem a gyorsgombokat, nyomjuk le a Ctrl + O -t, vagy Fájl → Open Folder.. és kattintsunk a feltölteni kívánt mappára 1x és alul mappaválasztás.

    Ezzel még csak megnyitottuk a weboldalt tartalmazó mappát, de a gépünkön repository a lokális gépen még nincsen.

    A meglévő fájlainkat úgy tudjuk hozzáadni a repository-hoz, hogy inicializáljuk a repository-t. 

    	git init

    Ha létre akarunk hozni egy README fájlt akkor:

    echo "# readyonhub.github.io" >> README.md

    Az idéző jelek közé írjuk a fájl tartalmát, a >> balra muató kacsacsőrök jelentik, hogy írja bele ezt a szöveget a README.md fájlba.

    Ha csak egy fájlt akarunk létrehozni vagy hozzáadni a repository-nkhoz, akkor:

    git add README.md

    Ha az összeset hozzá akarjuk adni a repository-nkhoz, ami a mappában található, akkor:

    	git add --all

    Hogy hozzáadjuk a verziókövető rendszerünkhöz a feltöltendő fájlainkat, írjuk be:

    git commit -m "Első komitálásom"

    Ezt követően meg kell tanitani a lokális git repository-nak, hogy távolban a szerverre, ezt hol töltse fel:

    git remote add origin https://github.com/READYonHub/readyonhub.github.io.git

    Az ellenőrzéshez, hogy helyesen csináltuk-e, úgy tudjuk ellenőrizni:

    git remote --verbose

    Itt ki listáza az ismert remote lokációkat, és megkell egyeznie a létrehozott repository URL-jével.

    Végül a helyi módosításainkat fel kell töltenünk a szerverre:

    git push -u origin main

    Kérheti a Username és Password-ünket, begéepljük a GitHubos adatainkat.

    Ha jól adtuk meg akkor csak ellenőrzésképp frissítsünk rá a GitHubos repository-nkra, és látnunk kell a feltöltött fájlainkat.

    A feltöltött weboldalunkat pedig úgy tekinthetjük meg, hoyg bemásoljuk a repository-nknak a nevét a böngészőbe:

    readyonhub.github.io
